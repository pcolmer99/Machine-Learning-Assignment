Now it’s cleanup time.  The following code loops around each column and determines whether there are more than 20% NAs in the data.  If there are, the column is removed from the training set.  Columns with too many NAs are useless as predictors.  Imputing does not help, as you need sufficient data to make a predictor useful.  I figured 20% was a good rule of thumb.

> totcol <- ncol(training)

> totrow <-nrow(training)

> remove_na <- c(0)

> for(i in 1:totcol) {

+     testna <- training[ ,i]

+     if (length(testna)*0.80 <= length(testna[is.na(testna)])) {

+         remove_na[length(remove_na)+1] <- i

+     }

+ }

> training <- training[ ,-remove_na]

I’m now ready to slice up my data.  I use only the training set and partition into xtraining and vtest variables.  Xtraining will be used to train my model, with vtest used for cross-validation.  I’m using a 60/40 split as recommended by Professor Leek.  My plan is to reserver the testing set, once I’ve done my cross-validation.  J

 

> inTrain <- createDataPartition(y=training$classe, p=0.6, list=FALSE)

> xtraining <- training[inTrain,]

> vtest <- training[-inTrain,]

 

Let’s go and train some models.  I’m going to train 3 models to see what the out of sample error is on the cross-validation set.  I could check for NA’s (Inputing) and check to see whether any variables are skewed (Standardisation), but I elect to experiment with the train settings first and observe the results.  I can also circle back if I’m not happy with my training and cross-validation results.

 

> modfit <- train(classe ~., method="rf", data=xtraining)

> modfit2 <- train(classe ~., method="rf", preProcess=c("center", "scale", "knnImpute"), data=xtraining)

> modfit3 <- train(classe ~., method="rf", trControl=trainControl(method = "cv", number = 3), data=xtraining)

So…I’m using random forest, as I believe it will give me the best prediction for the data we have.  I elect not to combine different models at this stage, however I may revisit this decision is the cross-validation accuracy results are poor.

 

I decide to use all the remaining predictors (52), except the classe variable to train the model.  It takes around 60 minutes to complete.

 

I’m now going to apply the prediction to the validation set and display only the accuracy.

 

> pred1 <- predict(modfit, newdata=vtest)

> confusionMatrix(pred1, vtest$classe)$overall[1]

 Accuracy

0.9963038

 

Next is modfit2, which is using 2 pre-processing techniques: standardising (to normalise skewed variables) and inputing (to remove NAs and resample).  I’m pretty confident that this won’t help, given my analysis , but I wanted to see what would happen in the accuracy.  This also takes around 60 minutes to run.

 

> pred2 <- predict(modfit2, newdata=vtest)

> confusionMatrix(pred2, vtest$classe)$overall[1]

 Accuracy

Less accuracy as expected, but not by much, so I can disregard this model.  Next is modfit3.  This time I’m using K-Fold Cross Validation.

 

pred3 <- predict(modfit3, newdata=vtest)

> confusionMatrix(pred3, vtest$classe)$overall[1] 

 Accuracy

0.9968137

 

Accuracy is higher but it only takes 5 minutes to run.  Something big in the processing has changed. 

 

BIG LEARNING:  When you run the train command without trControl settings, it defaults to bootstrap sampling.  By using K-Fold Cross Validation with number=3, you’re reducing the sampling and removing the repeats.  With this knowledge, I’ve decided that modfit is a better model.

 

So I use modfit and calculate the out of sample as follows:

 

> pred1 <- predict(modfit, newdata=vtest)

> 1 - confusionMatrix(pred1, vtest$classe)$overall[1]

   Accuracy

0.003696151

 

So my out of sample error = 0.37%
